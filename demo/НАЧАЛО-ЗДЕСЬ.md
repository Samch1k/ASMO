## 📌 С чего начать

### Вариант 1: Быстрый старт (30 минут)

```bash

cd /path/to/ASMO
pnpm install && pnpm build

# 2. Установите Claude CLI (для $0 режима)
```

## 🎬 Три демо-сценария

### Сценарий 1: YOLO Mode — Нулевое трение (5 мин)

**Что показываем:**
- Простая задача (сложность < 30) → автоматическое одобрение
- Без контрольных точек → выполнение < 60 секунд
- Устранение усталости от одобрений для ~20% задач

**Команды:**
```bash
asmo suggest "Исправить опечатку в README"
time asmo run "Исправить опечатку в README" --verbose
```

**Ключевой месседж:** ASMO не добавляет overhead для простых задач

---

### Сценарий 2: Context Cascade — Без потери контекста (10 мин)

**Что показываем:**
- Архитектор → создаёт стратегию валидации
- Разработчик → получает стратегию, реализует
- Тестер → получает стратегию + реализацию, тестирует

**Команды:**
```bash
asmo run "Добавить валидацию email в форму регистрации" --dry-run
asmo run "Добавить валидацию email в форму регистрации" --verbose
```

**Ключевой месседж:** Автоматическая передача контекста, без дублирования работы

---

### Сценарий 3: Party Mode — Параллельная коллаборация (10 мин)

**Что показываем:**
- Сложность ≥ 60 → активация Party mode
- 3+ агента работают параллельно
- Раунд 1: сходство 65% (ниже порога 80%)
- Раунд 2-3: сходство 87% → консенсус достигнут ✅

**Команды:**
```bash
asmo analyze "Реализовать OAuth2 аутентификацию с аудитом безопасности"
asmo run "Реализовать OAuth2 аутентификацию с аудитом безопасности" --verbose
```

**Ключевой месседж:** Интеллект через консенсус, не просто последовательное выполнение

## 💡 5 главных тезисов

### 1. **$0 стоимость с Session Mode**
- Claude CLI (Session mode) = $0
- Экономия $1,023/месяц vs API mode
- Команда из 10 человек = $10,230/месяц экономии

### 2. **YOLO Mode — Zero Friction**
- ~20% задач тривиальные (сложность < 30)
- Автоматическое одобрение
- Нет усталости от одобрений

### 3. **Context Cascade — No Context Loss**
- Автоматическая передача контекста между фазами
- Каждый агент строит на предыдущей работе
- Не требует конфигурации

### 4. **Party Mode — Intelligence через Consensus**
- Параллельное выполнение (не последовательное)
- Механизм конвергенции (порог 80%)
- Множественные перспективы снижают слепые зоны

### 5. **Adaptive Phase Detection — Smart, not Rigid**
- Вход в воркфлоу с любой фазы
- LLM анализирует существующую работу
- Экономит время и токены

## ❓ Частые вопросы

### Технические

**В: Что если Claude CLI не работает во время демо?**
О: ASMO автоматически переключится на API mode. Подготовьте `ANTHROPIC_API_KEY` в `.env` как backup.

**В: Что если сценарий не работает?**
О: Используйте backup plan: показывайте записанное видео или объясните на примере выводов.

**В: Сколько времени занимает каждый сценарий?**
О: Сценарий 1 (5 мин), Сценарий 2 (10 мин), Сценарий 3 (10 мин). Всего ~25 мин на live demo.

**В: Нужно ли показывать код?**
О: Опционально. Для технической аудитории — да (показать BaseAgent). Для менеджеров — нет.

### Интеграция

**В: Как интегрировать в наш CI/CD?**
О: CLI + exit codes. Примеры в `demo-presentation-outline.ru.md` секция 13.

**В: Поддерживает ли наш язык (Java/Go/Rust)?**
О: Да, ASMO language-agnostic. Работает с любой кодовой базой.

**В: Можно ли кастомизировать воркфлоу?**
О: Да, JSON шаблоны в `~/.asmo/config/workflows/`.
