## 🚀 Быстрый старт

### 1. Подготовка окружения

```bash
# Установка ASMO
cd /path/to/ASMO
pnpm install
pnpm build

# Установка Claude CLI (для Session mode, $0 стоимость)
brew install anthropic/claude/claude
claude auth login

# Проверка настройки
asmo suggest "тестовая задача"
```

### 2. Запуск демо-сценариев

**Вариант A: Интерактивное меню**
```bash
cd demo/
chmod +x demo-scenarios.ru.sh
./demo-scenarios.ru.sh
# Выберите сценарии 1-5
```

**Вариант B: Ручное выполнение**
```bash
# Сценарий 1: YOLO Mode (5 мин)
asmo suggest "Исправить опечатку в README"
asmo run "Исправить опечатку в README" --verbose

# Сценарий 2: Context Cascade (10 мин)
asmo run "Добавить валидацию email в форму регистрации" --dry-run
asmo run "Добавить валидацию email в форму регистрации" --verbose

# Сценарий 3: Party Mode (10 мин)
asmo analyze "Реализовать OAuth2 аутентификацию с аудитом безопасности"
asmo run "Реализовать OAuth2 аутентификацию с аудитом безопасности" --verbose
```

### 3. Проведение презентации

Используйте **demo-presentation-outline.ru.md** как гид:
- Следуйте структуре из 5 секций
- Используйте **slides.ru.md** как визуальную опору
- Запускайте сценарии live (или показывайте записи)

## 🎬 Демо-сценарии

### Сценарий 1: YOLO Mode — Нулевое трение (5 мин)

**Цель:** Показать авто-одобрение для тривиальных задач

**Команды:**
```bash
asmo suggest "Исправить опечатку в README"
time asmo run "Исправить опечатку в README" --verbose
```

**Ключевые моменты:**
- Сложность < 30 → YOLO mode
- Без контрольной точки одобрения
- Выполнение < 60 секунд
- Устранение усталости от одобрений

---

### Сценарий 2: Context Cascade — Без потери контекста (10 мин)

**Цель:** Показать передачу контекста между фазами

**Команды:**
```bash
asmo run "Добавить валидацию email в форму регистрации" --dry-run
asmo run "Добавить валидацию email в форму регистрации" --verbose
```

**На что обратить внимание в verbose выводе:**
- Фаза 1 (Архитектор): Создаёт стратегию валидации
- Фаза 2 (Разработчик): Получает стратегию, реализует
- Фаза 3 (Тестер): Получает стратегию + реализацию, тестирует

**Ключевые моменты:**
- Автоматическое распространение контекста
- Без дублирования решений
- Каждый агент строит на предыдущей работе

---

### Сценарий 3: Party Mode — Параллельная коллаборация (10 мин)

**Цель:** Показать параллельное выполнение с консенсусом

**Команды:**
```bash
asmo analyze "Реализовать OAuth2 аутентификацию с аудитом безопасности"
asmo run "Реализовать OAuth2 аутентификацию с аудитом безопасности" --verbose
```

**На что обратить внимание в verbose выводе:**
- Сложность ≥ 60 → активация Party mode
- Раунд 1: Несколько агентов работают параллельно
- Раунды 2-3: Проверки конвергенции
- Достигнут консенсус (сходство > 80%)

**Ключевые моменты:**
- Параллельное vs последовательное выполнение
- Механизм конвергенции
- Множественные перспективы снижают слепые зоны
- 3x затраты токенов, компенсируются Session mode ($0)

## 💡 Тезисы для обсуждения

### Подчеркните эти фичи

1. **$0 стоимость с Session Mode**
   - Session mode (Claude CLI) по умолчанию
   - Экономия $1,023/месяц vs API mode (для 100 задач/день)
   - Команда из 10 человек: экономия $10,230/месяц

2. **YOLO Mode**
   - ~20% задач тривиальные
   - Авто-одобрение (без запроса человеку)
   - Нулевая усталость от одобрений

3. **Context Cascade**
   - Автоматический, не ручной
   - Не требует конфигурации
   - Каждый агент видит полную историю

4. **Party Mode**
   - Интеллект, а не просто последовательное выполнение
   - Механизм консенсуса
   - Качество через разнообразие

5. **Adaptive Phase Detection**
   - Умный, а не жёсткий
   - Возобновление с существующей работы
   - Экономия времени и токенов

## ❓ Частые вопросы и ответы

**В: Что делать, если Claude CLI недоступен во время демо?**
О: ASMO автоматически переключится на API mode. Имейте `ANTHROPIC_API_KEY` в `.env` как запасной вариант.

**В: Что делать, если сценарий не сработал?**
О: Используйте запасной план (записанное видео или примеры выводов). Объясните, что должно было произойти.

**В: Сколько должно занять полное демо?**
О: 45-60 минут включая Q&A. Корректируйте в зависимости от вовлечённости аудитории.

**В: Можно ли пропустить сценарии?**
О: Да! Минимум: Покажите Сценарий 1 (YOLO) + 1 из (Сценарий 2 или 3). Полное демо: все 3.

**В: Стоит ли показывать код?**
О: Опционально. Для технической аудитории показ кода агентов ценен. Для остальных — фокус на возможностях.
