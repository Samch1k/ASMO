## Слайд 1: Титульный

**ASMO**
**AI System for Multiagent Orchestration**

*Интеллектуальная маршрутизация задач, мультиагентные воркфлоу, режим LLM с нулевой стоимостью*

**Длительность демонстрации:** 45-60 минут
**Аудитория:** Техническая команда

## Слайд 2: Проблема

### Текущие вызовы AI-разработки

| Проблема | Влияние |
|---------|---------|
| 💰 **Высокие затраты на API LLM** | Ограничивает внедрение, счета $1000+/месяц |
| 🎯 **Ручная маршрутизация задач** | Разработчик решает, какой инструмент/агент использовать |
| 🔗 **Потеря контекста между агентами** | Повторяющаяся работа, пропущенные решения |
| ⏳ **Усталость от подтверждений** | Простые задачи блокируются контрольными точками |
| 🤖 **Ограничения одного агента** | Одна перспектива, потенциальные слепые зоны |

**Пример:**
```
Воркфлоу разработчика сегодня:
1. Выбрать правильный AI инструмент для задачи (ChatGPT? Copilot? Claude?)
2. Копировать контекст вручную между сессиями
3. Подтверждать каждое изменение (даже опечатки)
4. Платить $20-50/месяц на разработчика
```

## Слайд 3: Решение ASMO

### Интеллектуальная мультиагентная оркестрация

```
┌───────────────────────────────────────────────────────┐
│  "Исправить null pointer в обработчике логина"        │
│                         ↓                              │
│  [ComplexityAnalyzer] → score: 35 (simple, bug_fix)   │
│                         ↓                              │
│  [WorkflowSelector] → bug_fix_workflow                │
│                         ↓                              │
│  [WorkflowEngine] → debugger → developer → tester     │
│                         ↓                              │
│  Результат: Исправление применено, тесты добавлены, 12 минут│
└───────────────────────────────────────────────────────┘
```

**Ключевые цифры:**
- **25 агентов** (основные, специализированные, валидация)
- **27 воркфлоу** (от тривиальных до корпоративных)
- **55 навыков** (централизованная библиотека)
- **$0/месяц** (с режимом Session)

## Слайд 4: Архитектура — 5-этапный конвейер

```
ВВОД → АНАЛИЗ → МАРШРУТИЗАЦИЯ → ВЫПОЛНЕНИЕ → РЕЗУЛЬТАТ
```

### Этап 1: Ввод задачи
```bash
asmo run "Implement OAuth2 authentication"
```

### Этап 2: Анализ сложности
```json
{
  "score": 75,
  "level": "complex",
  "categories": ["security", "feature", "api"],
  "impacts": ["security_impact", "high_risk"]
}
```

### Этап 3: Выбор воркфлоу
```
Соответствие: feature_implementation_full
Агенты: architect → security-specialist → developer → tester
Режим: party (сложность ≥ 60)
```

### Этап 4: Выполнение
```
Фаза 1: Architect (дизайн OAuth2 потока)
  ↓ ContextCascade
Фаза 2: Security Specialist (модель угроз)
  ↓ ContextCascade
Фаза 3: Developer (реализация)
  ↓ ContextCascade
Фаза 4: Tester (тесты безопасности)
```

### Этап 5: Результат
```

✓ Тесты: 15 созданы, все проходят
```

## Слайд 7: Уникальная функция #1 — Режим YOLO

### Проблема
Тривиальные задачи (исправление опечаток, изменения текста) не должны требовать подтверждения

### Решение
Авто-подтверждение задач со сложностью < 30

### Как это работает
```typescript
if (complexity.score < 30) {
  // Режим YOLO активирован
  ApprovalCheckpoint.bypass()
  // Выполнить немедленно, без запроса человеку
}
```

### Пример
```bash
asmo run "Fix typo in README"

# Вывод:
# [ComplexityAnalyzer] Оценка: 15 (тривиальная)
# [YOLO Mode] Авто-подтверждение (оценка < 30)
# [Developer] Исправляю опечатку...
# ✓ Завершено за 45 секунд
```

### Влияние
- **~20% задач** тривиальны
- **Нулевая усталость от подтверждений** для простых изменений
- **< 60 секунд** время выполнения

## Слайд 8: Уникальная функция #2 — Каскад контекста

### Проблема
Агенты теряют контекст между фазами → повторяющаяся работа

### Решение
Автоматический поток данных между фазами

### Как это работает
```typescript
Фаза 1: Architect создает дизайн
  artifacts = { validationStrategy: "клиент + сервер" }

Фаза 2: Developer получает контекст Фазы 1
  input.context = Phase1.artifacts
  // Developer видит: архитектор решил клиент + сервер
  // Не нужно переопределять стратегию

Фаза 3: Tester получает контекст Фазы 1 + Фазы 2
  input.context = Phase1.artifacts + Phase2.artifacts
  // Tester видит: стратегию + реализацию
  // Создает тесты для клиента и сервера
```

### Преимущества
- 🚫 **Нет дублирования** — решения принимаются один раз
- 📦 **Автоматический** — не требуется конфигурация
- 📝 **Накопительный** — последующие фазы видят полную историю
- 🔗 **Непрерывность** — плавный воркфлоу

## Слайд 9: Уникальная функция #3 — Party Mode

### Проблема
Сложные задачи нуждаются в множественных перспективах

### Решение
Параллельное мультиагентное сотрудничество с консенсусом

### Когда активируется
- Сложность ≥ 60
- Обнаружено множество паттернов
- Кросс-секционные проблемы (безопасность + производительность)

### Как это работает
```
Раунд 1: architect + security-specialist + developer (параллельно)
  → 3 независимых анализа

Раунд 2: Проверка конвергенции
  → Схожесть = 65% (ниже порога 80%)
  → Продолжить обсуждение

Раунд 3: Проверка конвергенции
  → Схожесть = 87% (выше порога 80%) ✅
  → Консенсус достигнут!
```

### Механизм конвергенции
```typescript
similarity = calculateSimilarity(
  agentOutputs[round],
  agentOutputs[round - 1]
)

if (similarity >= 0.8) {
  return consolidatedResult  // Консенсус!
}
```

### Компромиссы
- ✅ Лучшее качество (множественные точки зрения)
- ✅ Снижение слепых зон
- ❌ 3x использование токенов (смягчается режимом Session $0)

## Слайд 10: Уникальная функция #4 — Адаптивное определение фазы

### Проблема
Начало воркфлоу с нуля, когда работа уже существует

### Решение
LLM анализирует существующий контекст, присоединяется к воркфлоу посередине

### Пример
```
Сценарий: Вы уже исследовали баг вручную

bug_fix_workflow фазы:
  Фаза 1: Исследование (debugger)
  Фаза 2: Исправление (developer)
  Фаза 3: Тестирование (tester)

У вас есть: investigation-report.md

PhaseDetector анализирует:
  → "investigation-report.md содержит анализ корневой причины"
  → "Фаза 1 (исследование) завершена"
  → "Присоединиться к воркфлоу на Фазе 2 (developer)"
  → Метаданные: { detectedPhase: 2, skippedPhases: [1] }

Воркфлоу начинается с Фазы 2:
  ✓ Сэкономлено время (нет избыточного исследования)
  ✓ Сэкономлены токены LLM (Фаза 1 пропущена)
  ✓ Строится на существующей работе
```

### Сценарии использования
- Возобновление прерванных воркфлоу
- Присоединение к существующим проектам
- Работа частично завершена

## Слайд 12: Метрики и обучение

### Как это работает

**После каждого выполнения:**
```typescript
1. MetricsCollector записывает:
   - Использованный воркфлоу
   - Оценка сложности
   - Продолжительность, использование токенов
   - Успех/неудача
   - Производительность агента

2. MetricsPersister сохраняет в SQLite

3. MetricsOptimizer учится:
   - Лучший воркфлоу для каждого диапазона сложности
   - Средняя продолжительность для оценки
   - Паттерны эффективности токенов

4. Будущая маршрутизация использует изученные данные
```

### Пример метрик
```bash
asmo metrics --workflow bug_fix_workflow

# Вывод:
# Воркфлоу: bug_fix_workflow
# Выполнений: 47
# Уровень успеха: 95.7%
# Средняя сложность: 32 (простая)
# Средняя продолжительность: 11м 34с
# Средние токены: 4,200
#
# Рекомендация: Оптимально для простых исправлений багов (оценка 21-40)
```

### Преимущества
- 📈 Непрерывное улучшение
- 🎯 Лучшая маршрутизация со временем
- ⏱️ Точные оценки
- 💰 Оптимизация токенов

```bash

if [ "$(echo $SUGGESTION | jq -r '.useAsmo')" = "true" ]; then
```

## Слайд 17: Вопросы и ответы — Частые вопросы

### Технические
**В:** Что если агент не работает?
**О:** CircuitBreaker изолирует сбой, плавная деградация

**В:** Можем ли мы добавить кастомных агентов?
**О:** Да, расширьте BaseAgent, определите JSON роли + навыки

**В:** Насколько точен режим эвристик?
**О:** ~65%, хорошо для оффлайн/быстрых проверок

**В:** Использование токенов для средней задачи?
**О:** Простые: 2-4K, Средние: 5-10K, Сложные: 15-20K

### Интеграция
**В:** Работает с нашим CI/CD?
**О:** Да, CLI + exit коды

**В:** Поддержка Java/Go/Rust?
**О:** Да, не зависит от языка

**В:** Кастомизация воркфлоу?
**О:** Да, JSON шаблоны в `~/.asmo/config/`

### Стоимость
**В:** Claude CLI недоступен?
**О:** Авто-переход на режим API

**В:** Ограничить использование токенов?
**О:** Да, настройте макс раундов для party mode

**В:** Предпросмотр стоимости перед выполнением?
**О:** Да, `--dry-run` показывает план + оценку токенов
