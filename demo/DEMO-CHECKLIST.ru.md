# ASMO Техническая Демонстрация — Контрольный Список Подготовки

Выполните этот контрольный список перед проведением демонстрации ASMO для обеспечения плавного выполнения.

---

## Подготовка перед демонстрацией (За день до)

### Настройка окружения
- [ ] **Установите зависимости ASMO**
  ```bash
  cd /path/to/ASMO
  pnpm install
  pnpm build
  ```

- [ ] **Установите и аутентифицируйте Claude CLI** (для режима Session)
  ```bash
  brew install anthropic/claude/claude
  claude auth login
  # Проверка: claude --version
  ```

- [ ] **Проверьте работу режима Session**
  ```bash
  asmo suggest "test task"
  # Должен вернуть JSON без требования API-ключа
  ```

- [ ] **Опционально: Настройте резервный API** (если демонстрируете переключение)
  ```bash
  echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env
  ```

### Демонстрационные материалы
- [ ] **Клонируйте/подготовьте демонстрационный проект**
  ```bash
  cd demo/sample-project
  # Проверьте наличие файлов:
  # - README.md (с намеренной опечаткой)
  # - src/components/SignupForm.tsx
  ```

- [ ] **Протестируйте все демонстрационные сценарии** (запустите хотя бы один раз)
  ```bash
  cd demo/
  chmod +x demo-scenarios.sh
  ./demo-scenarios.sh
  # Запустите сценарии 1, 2, 3 и бонусный
  ```

- [ ] **Проверьте результаты демонстрации**
  - Сценарий 1 выполняется за < 60с
  - Сценарий 2 показывает каскад контекста в verbose
  - Сценарий 3 активирует party mode (оценка ≥ 60)
  - Бонус показывает определение фазы

- [ ] **Просмотрите слайды**
  ```bash
  # Прочитайте demo/slides.md
  # Убедитесь, что все примеры соответствуют текущему поведению ASMO
  ```

### Техническая подготовка
- [ ] **Настройка терминала**
  - [ ] Крупный размер шрифта (видимый с задних рядов)
  - [ ] Подсветка синтаксиса включена
  - [ ] Тема терминала: высокий контраст (предпочтительно темный фон)
  - [ ] Размер окна: полноэкранный или готовый к разделению

- [ ] **Резервная запись экрана**

  ```bash
  # На случай сбоя живой демонстрации, проиграйте записи
  ```

- [ ] **tmux/мультиплексор терминала** (опционально)
  ```bash

  # - Выполнения команд (слева)
  # - Вывода логов (справа)
  # - Слайдов/заметок (внизу)
  ```

## Во время демонстрации

### Введение (5 мин)
- [ ] Начните со слайда формулировки проблемы
- [ ] Покажите диаграмму архитектуры ASMO
- [ ] Подчеркните ключевые цифры: 25 агентов, 27 воркфлоу, стоимость $0

### Сценарий 1: Режим YOLO (5 мин)
- [ ] Команда: `asmo suggest "Fix typo in README"`
- [ ] Покажите вывод JSON (useAsmo: true, score: 15)
- [ ] Команда: `time asmo run "Fix typo in README" --verbose`
- [ ] Подчеркните: активация YOLO, выполнение < 60с
- [ ] **Ключевые моменты:**
  - Оценка < 30 → авто-подтверждение
  - Устраняет усталость от подтверждений
  - ~20% задач тривиальны

### Сценарий 2: Каскад контекста (10 мин)
- [ ] Команда: `asmo run "Add email validation to signup form" --dry-run`
- [ ] Покажите план выполнения (3 фазы)
- [ ] Команда: `asmo run "Add email validation to signup form" --verbose`
- [ ] **Отметьте в verbose выводе:**
  - Фаза 1: Архитектор создает стратегию
  - Фаза 2: Разработчик получает стратегию, реализует
  - Фаза 3: Тестировщик получает стратегию + реализацию
- [ ] **Ключевые моменты:**
  - Автоматический поток контекста
  - Нет дублирования работы
  - Каждый агент строит на предыдущих

### Сценарий 3: Party Mode (10 мин)
- [ ] Команда: `asmo analyze "Implement OAuth2 authentication with security audit"`
- [ ] Покажите сложность: 75, уровень complex
- [ ] Команда: `asmo run "Implement OAuth2 authentication with security audit" --verbose`
- [ ] **Отметьте в verbose выводе:**
  - Активация party mode (оценка ≥ 60)
  - Несколько агентов параллельно
  - Проверки конвергенции (Раунд 1: 65%, Раунд 3: 87%)
  - Достигнут консенсус
- [ ] **Ключевые моменты:**
  - Параллельный vs последовательный
  - Механизм конвергенции
  - Множественные перспективы снижают слепые зоны

### Бонус: Адаптивное определение фазы (5 мин, опционально)
- [ ] Создайте investigation-report.md вручную
- [ ] Команда: `asmo run "Fix null pointer in login handler" --verbose`
- [ ] Покажите: PhaseDetector пропускает Фазу 1, присоединяется к Фазе 2
- [ ] **Ключевые моменты:**
  - Возобновляет с существующей работы
  - Экономит время и токены
  - Умный, не жесткий

### Уникальные функции (10 мин)
- [ ] Просмотрите слайды для:
  - Dual LLM Strategy
  - Метрики и цикл обучения
  - Концепция Adversarial Review
  - Сравнение стоимости ($0 vs $1,023/месяц)

### Вопросы и ответы (10 мин)

  - "Можем ли мы настраивать воркфлоу?"

## Советы по презентации

### Делайте
✅ Говорите медленно и четко (техническая аудитория, термины важны)
✅ Объясняйте, что вы печатаете, перед выполнением
✅ Делайте паузу после каждой команды, чтобы отобразился вывод
