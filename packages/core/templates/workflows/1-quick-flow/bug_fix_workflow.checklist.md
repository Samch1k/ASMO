# Bug Fix Workflow Checklist

## Phase 1: Investigation (debugger)

### Before Starting
- [ ] Bug report is clear and reproducible
- [ ] Error logs/stack traces are available
- [ ] Environment details are documented

### During Execution
- [ ] Root cause identified with evidence
- [ ] Related code areas examined
- [ ] Impact scope assessed

### Completion Criteria
- [ ] Root cause clearly documented
- [ ] Fix approach proposed
- [ ] No additional issues discovered

## Phase 2: Fix Implementation (developer)

### Before Starting
- [ ] Root cause analysis reviewed
- [ ] Fix approach approved
- [ ] Tests identified for validation

### During Execution
- [ ] Fix implemented following code standards
- [ ] No breaking changes introduced
- [ ] Related documentation updated

### Completion Criteria
- [ ] Code compiles without errors
- [ ] Fix addresses root cause
- [ ] No new bugs introduced

## Phase 3: Verification (tester)

### Before Starting
- [ ] Test cases prepared
- [ ] Test environment ready
- [ ] Acceptance criteria defined

### During Execution
- [ ] All test cases executed
- [ ] Edge cases validated
- [ ] Regression testing performed

### Completion Criteria
- [ ] All tests passing
- [ ] Bug verified as fixed
- [ ] No regressions detected

## Final Approval

- [ ] All phases completed successfully
- [ ] Code reviewed and approved
- [ ] Documentation updated
- [ ] Ready for deployment
