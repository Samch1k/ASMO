# Performance Investigation Checklist

## Phase 1: Debug Analysis (debugger)

### Before Starting
- [ ] Performance issue documented
- [ ] Reproduction steps clear
- [ ] Baseline metrics recorded

### During Execution
- [ ] Profiling tools configured
- [ ] Bottlenecks identified
- [ ] Call stacks analyzed
- [ ] Memory usage profiled

### Completion Criteria
- [ ] Root cause identified
- [ ] Performance metrics collected
- [ ] Bottleneck locations documented
- [ ] Fix approach proposed

## Phase 2: Optimization Analysis (performance-optimizer)

### Before Starting
- [ ] Debug report reviewed
- [ ] Optimization targets set
- [ ] Tools configured

### During Execution
- [ ] Database queries analyzed
- [ ] Caching opportunities identified
- [ ] Algorithm complexity assessed
- [ ] Resource usage profiled

### Completion Criteria
- [ ] Optimization opportunities documented
- [ ] Expected improvements estimated
- [ ] Implementation plan created
- [ ] Trade-offs analyzed

## Phase 3: Fix Implementation (developer)

### Before Starting
- [ ] Analysis reports reviewed
- [ ] Optimization priorities set
- [ ] Testing strategy defined

### During Execution
- [ ] Performance fixes implemented
- [ ] Caching added (if applicable)
- [ ] Queries optimized
- [ ] Code refactored

### Completion Criteria
- [ ] All optimizations applied
- [ ] Code follows standards
- [ ] No functionality broken
- [ ] Tests updated

## Phase 4: Validation Testing (tester)

### Before Starting
- [ ] Performance baselines ready
- [ ] Test scenarios prepared
- [ ] Monitoring tools configured

### During Execution
- [ ] Performance tests run
- [ ] Before/after comparison made
- [ ] Load tests executed
- [ ] Edge cases validated

### Completion Criteria
- [ ] Performance improved
- [ ] Target metrics achieved
- [ ] No regressions introduced
- [ ] System stable

## Final Approval

- [ ] All phases completed successfully
- [ ] Performance gains measured
- [ ] Documentation updated
- [ ] Ready for deployment
