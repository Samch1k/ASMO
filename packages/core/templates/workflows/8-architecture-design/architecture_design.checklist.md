# Architecture Design Workflow Checklist

## Phase 1: Requirements Analysis (architect)

### Before Starting
- [ ] Business requirements documented
- [ ] Stakeholders identified
- [ ] Current system understood (if redesign)

### During Execution
- [ ] Functional requirements documented
- [ ] Non-functional requirements identified
- [ ] Quality attributes prioritized (scalability, performance, security, etc.)
- [ ] Constraints documented (budget, time, technology, etc.)
- [ ] Success criteria defined

### Completion Criteria
- [ ] Requirements document complete
- [ ] Constraints clearly identified
- [ ] Quality attributes prioritized
- [ ] Stakeholder approval obtained

## Phase 2: High-Level Design (architect)

### Before Starting
- [ ] Requirements reviewed
- [ ] Similar systems researched
- [ ] Technology options evaluated

### During Execution
- [ ] System components identified
- [ ] Component responsibilities defined
- [ ] Interaction patterns designed
- [ ] Technology stack selected
- [ ] Architectural patterns chosen
- [ ] System architecture diagram created

### Completion Criteria
- [ ] High-level architecture complete
- [ ] Component diagram documented
- [ ] Technology stack justified
- [ ] Patterns documented
- [ ] Initial review passed

## Phase 3: Detailed Design (architect)

### Before Starting
- [ ] High-level design approved
- [ ] Component interfaces defined

### During Execution
- [ ] API contracts specified
- [ ] Data models designed
- [ ] Sequence diagrams created
- [ ] Deployment architecture designed
- [ ] ADRs (Architecture Decision Records) written
- [ ] Security architecture defined
- [ ] Error handling strategy defined

### Completion Criteria
- [ ] All APIs documented
- [ ] Data models complete
- [ ] Sequence diagrams for key flows
- [ ] Deployment architecture documented
- [ ] All major decisions recorded in ADRs

## Phase 4: Review (code-reviewer)

### Before Starting
- [ ] All design documents complete
- [ ] Review criteria established

### During Execution
- [ ] Scalability reviewed
- [ ] Performance characteristics assessed
- [ ] Security architecture evaluated
- [ ] Maintainability assessed
- [ ] Risks identified and documented
- [ ] Improvement opportunities noted

### Completion Criteria
- [ ] Architecture review report complete
- [ ] All concerns addressed or documented
- [ ] Risk mitigation strategies defined
- [ ] Architecture approved

## Phase 5: Infrastructure Planning (devops)

### Before Starting
- [ ] Architecture design reviewed
- [ ] Cloud/on-premise strategy decided

### During Execution
- [ ] Infrastructure requirements defined
- [ ] Deployment strategy designed
- [ ] CI/CD pipeline planned
- [ ] Monitoring and logging planned
- [ ] Backup and disaster recovery planned
- [ ] Cost estimation completed

### Completion Criteria
- [ ] Infrastructure plan complete
- [ ] Deployment strategy documented
- [ ] Monitoring plan defined
- [ ] Disaster recovery plan ready
- [ ] Costs estimated and approved

## Final Approval

- [ ] All phases completed successfully
- [ ] Complete architecture documentation
- [ ] All ADRs written and approved
- [ ] Infrastructure plan approved
- [ ] Stakeholder sign-off obtained
- [ ] Ready for implementation
