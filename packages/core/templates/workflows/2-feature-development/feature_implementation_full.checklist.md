# Full Feature Implementation Checklist

## Phase 1: Design (architect)

### Before Starting
- [ ] Feature requirements documented
- [ ] User stories defined
- [ ] Technical constraints identified

### During Execution
- [ ] System architecture designed
- [ ] API contracts defined
- [ ] Data models specified
- [ ] Integration points identified

### Completion Criteria
- [ ] Architecture diagram created
- [ ] API specification complete
- [ ] Design review approved
- [ ] Technical feasibility confirmed

## Phase 2: Implementation (developer + frontend-engineer)

### Before Starting
- [ ] Design documents reviewed
- [ ] Development environment ready
- [ ] Dependencies identified

### During Execution
- [ ] Backend code implemented
- [ ] Frontend components built
- [ ] API endpoints tested
- [ ] Code follows project standards

### Completion Criteria
- [ ] All feature code complete
- [ ] Unit tests written and passing
- [ ] Code compiles without errors
- [ ] No lint/formatting issues

## Phase 3: Testing (tester)

### Before Starting
- [ ] Test plan prepared
- [ ] Test data ready
- [ ] Acceptance criteria clear

### During Execution
- [ ] Unit tests executed
- [ ] Integration tests run
- [ ] E2E tests performed
- [ ] Edge cases validated

### Completion Criteria
- [ ] All tests passing
- [ ] No critical bugs found
- [ ] Performance acceptable
- [ ] Feature works as expected

## Phase 4: Deployment (deployer)

### Before Starting
- [ ] All tests passed
- [ ] Deployment plan ready
- [ ] Rollback plan prepared

### During Execution
- [ ] Database migrations applied
- [ ] Code deployed to staging
- [ ] Smoke tests performed
- [ ] Production deployment executed

### Completion Criteria
- [ ] Feature live in production
- [ ] Monitoring configured
- [ ] No deployment errors
- [ ] Health checks passing

## Final Approval

- [ ] All phases completed successfully
- [ ] Feature documented
- [ ] Team notified
- [ ] Ready for users
