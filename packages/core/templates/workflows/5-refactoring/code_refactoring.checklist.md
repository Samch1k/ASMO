# Code Refactoring Workflow Checklist

## Phase 1: Analysis (architect)

### Before Starting
- [ ] Current codebase state documented
- [ ] Known issues and pain points identified
- [ ] Business requirements understood

### During Execution
- [ ] Code structure analyzed
- [ ] Design patterns evaluated
- [ ] Dependencies mapped
- [ ] Improvement opportunities identified
- [ ] Refactoring approach defined

### Completion Criteria
- [ ] Analysis report complete
- [ ] Refactoring scope clearly defined
- [ ] Risks identified and documented
- [ ] Approach approved

## Phase 2: Assessment (code-reviewer)

### Before Starting
- [ ] Analysis report reviewed
- [ ] Quality metrics baseline established

### During Execution
- [ ] Code quality issues prioritized
- [ ] Impact of each issue assessed
- [ ] Dependencies between issues identified
- [ ] Risk analysis completed

### Completion Criteria
- [ ] Prioritized list of refactoring targets
- [ ] Risk assessment documented
- [ ] Stakeholder approval obtained

## Phase 3: Test Preparation (tester)

### Before Starting
- [ ] Current functionality documented
- [ ] Test environment prepared

### During Execution
- [ ] Baseline tests written/updated
- [ ] Test coverage measured
- [ ] Regression test suite prepared
- [ ] Test automation configured

### Completion Criteria
- [ ] Comprehensive test suite in place
- [ ] All tests passing on current code
- [ ] Test coverage meets threshold
- [ ] Baseline metrics captured

## Phase 4: Refactoring (developer)

### Before Starting
- [ ] Refactoring plan reviewed
- [ ] Tests verified as passing
- [ ] Version control branch created

### During Execution
- [ ] Changes implemented incrementally
- [ ] Tests run after each change
- [ ] Code follows standards
- [ ] Comments and documentation updated
- [ ] No functionality changes made

### Completion Criteria
- [ ] All refactoring completed
- [ ] All tests still passing
- [ ] Code follows best practices
- [ ] Documentation updated
- [ ] No breaking changes introduced

## Phase 5: Validation (code-reviewer)

### Before Starting
- [ ] Refactored code ready for review
- [ ] Test results available

### During Execution
- [ ] Code quality improvements verified
- [ ] Functionality preservation confirmed
- [ ] Test coverage reviewed
- [ ] Documentation reviewed
- [ ] Performance impact assessed

### Completion Criteria
- [ ] Code quality measurably improved
- [ ] All functionality preserved
- [ ] Tests comprehensive and passing
- [ ] Documentation complete
- [ ] Approved for merge

## Final Approval

- [ ] All phases completed successfully
- [ ] Code quality metrics improved
- [ ] No functionality regressions
- [ ] Team reviewed and approved
- [ ] Ready for deployment
