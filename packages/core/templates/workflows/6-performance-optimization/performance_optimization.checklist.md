# Performance Optimization Workflow Checklist

## Phase 1: Profiling (optimizer)

### Before Starting
- [ ] Performance goals defined
- [ ] Profiling tools configured
- [ ] Baseline metrics captured
- [ ] Test environment prepared

### During Execution
- [ ] System profiled under realistic load
- [ ] CPU, memory, I/O metrics collected
- [ ] Slow queries/endpoints identified
- [ ] Resource usage analyzed
- [ ] Bottlenecks documented with evidence

### Completion Criteria
- [ ] Performance baseline established
- [ ] Bottlenecks clearly identified
- [ ] Metrics documented with numbers
- [ ] Impact prioritized

## Phase 2: Strategy (architect)

### Before Starting
- [ ] Profiling results reviewed
- [ ] Performance goals confirmed
- [ ] System architecture understood

### During Execution
- [ ] Optimization approaches evaluated
- [ ] Trade-offs analyzed
- [ ] Architecture improvements designed
- [ ] Implementation risks assessed
- [ ] Expected improvements estimated

### Completion Criteria
- [ ] Optimization strategy defined
- [ ] Expected improvements quantified
- [ ] Risks documented
- [ ] Approach approved

## Phase 3: Implementation (developer)

### Before Starting
- [ ] Optimization plan reviewed
- [ ] Performance tests prepared
- [ ] Development environment ready

### During Execution
- [ ] Optimizations implemented incrementally
- [ ] Performance tested after each change
- [ ] Code quality maintained
- [ ] Documentation updated
- [ ] No functionality broken

### Completion Criteria
- [ ] All optimizations implemented
- [ ] Performance improvements measured
- [ ] Code quality maintained
- [ ] Tests passing
- [ ] Documentation updated

## Phase 4: Validation (tester)

### Before Starting
- [ ] Optimized code ready for testing
- [ ] Test cases prepared
- [ ] Load testing tools configured

### During Execution
- [ ] Performance improvements verified
- [ ] Regression tests executed
- [ ] Load tests performed
- [ ] Edge cases validated
- [ ] Metrics compared to baseline

### Completion Criteria
- [ ] Performance goals achieved
- [ ] No functional regressions
- [ ] System stable under load
- [ ] Metrics meet targets
- [ ] Edge cases handled

## Phase 5: Monitoring (optimizer)

### Before Starting
- [ ] Optimization complete and validated
- [ ] Monitoring requirements defined

### During Execution
- [ ] Performance metrics dashboards created
- [ ] Alerts configured for degradation
- [ ] Baselines updated
- [ ] Documentation finalized
- [ ] Team trained on monitoring

### Completion Criteria
- [ ] Monitoring dashboards active
- [ ] Alerts configured and tested
- [ ] Optimization report complete
- [ ] Team trained
- [ ] Ready for production

## Final Approval

- [ ] All phases completed successfully
- [ ] Performance targets achieved
- [ ] No regressions detected
- [ ] Monitoring and alerts in place
- [ ] Documentation complete
- [ ] Ready for deployment
