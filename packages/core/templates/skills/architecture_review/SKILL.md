# Architecture Review

## Overview

Systematic review of system architecture covering design decisions, technology choices, scalability, and documentation quality.

## Process

1. Review architectural decisions and trade-offs
2. Evaluate technology choices against requirements
3. Assess system design for scalability and maintainability
4. Create or update Architecture Decision Records (ADRs)
5. Identify architectural risks and technical debt

## Combines Skills

- architecture_decisions
- system_design

- adr_creation

## When to Use

- Before major architectural changes
- During system design reviews
- When evaluating technical debt
- Quarterly architecture health checks

**Suitable for**: Medium to large systems
