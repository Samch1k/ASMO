## Common Patterns Across Examples

### 1. Workflow Structure
All examples follow the same pattern:
```
1. Check if ASMO is needed (asmo suggest)
2. Analyze complexity (automatic)
3. Review and approve workflow
4. Execute phases (agents work)
5. Review artifacts
6. Test and commit
```

### 2. Artifacts
Every workflow generates:
- **Documentation**: Analysis, design, implementation guides
- **Code**: Source files, tests, configurations
- **Reports**: Test results, metrics, recommendations

### 3. Time Estimates
ASMO provides accurate estimates:
- Simple: 1-2 hours
- Medium: 2-6 hours
- Complex: 6+ hours

### 4. Adaptive Features
Workflows adjust to complexity:
- Simple tasks skip optional steps
- Complex tasks include all steps
- Time savings: 20-40% for simple tasks

## Try These Examples

### Clone Example Repository
```bash
git clone https://github.com/asmo-examples/all-examples
cd all-examples
```

### Run Individual Examples
```bash
# Bug fix example
cd 01-bug-fix
asmo run "Fix case-sensitive email login"

# Feature example
cd 02-user-profile
asmo run "Add user profile page with avatar upload, name, email, and bio editing"

# Testing example
cd 03-testing
asmo workflow tea_planning_workflow --task "Plan comprehensive tests for checkout flow"

# Architecture example
cd 04-architecture
asmo workflow architecture_design --task "Design microservices architecture for e-commerce"

# Code review example
cd 05-code-review
asmo workflow code_review_workflow --task "Review PR #42: Add authentication"

# Security example
cd 06-security
asmo workflow security_audit --task "Audit payment processing code"
```
