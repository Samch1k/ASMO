# ASMO Framework: Обзор системы

> Документ для нетехнических специалистов

## Что это?

ASMO Framework — это интеллектуальная система управления разработкой, которая автоматически определяет сложность задач и подбирает оптимальный подход к их решению.

## Как это работает?

```
┌─────────────────────────────────────────────────────────────┐
│                    Поступает задача                          │
│              "Добавить кнопку выхода"                        │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              🎯 Анализ сложности (0-100)                     │
│                                                              │
│   Простая задача (0-30)  →  YOLO Mode (быстро, без проверок)│
│   Средняя (30-60)        →  Стандартный процесс             │
│   Сложная (60-100)       →  Полное планирование + ревью     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              👥 Подбор специалистов                          │
│                                                              │
│   • Разработчик      • Архитектор    • Тестировщик          │
│   • UI-специалист    • DevOps        • Безопасник           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              ✅ Выполнение и контроль                        │
│                                                              │
│   Отслеживание прогресса → Автоматические проверки → Готово │
└─────────────────────────────────────────────────────────────┘
```

## Ключевые преимущества

| Было раньше | Стало с ASMO |
|-------------|---------------|
| Все задачи проходили одинаковый процесс | Умный подбор процесса под сложность |
| Ручное планирование каждой задачи | Автоматические рекомендации |
| Легко забыть про задачу | Централизованный трекинг всех задач |
| Споры о подходах затягивались | Структурированный мозговой штурм с голосованием |

## Пять главных компонентов

### 1. 🚀 YOLO Mode (You Only Live Once)

**Для кого:** Простые задачи со сложностью менее 30 баллов.

**Что делает:** Позволяет выполнять тривиальные задачи (исправить опечатку, обновить текст) мгновенно, без лишних согласований и проверок.

**Пример:** Задача "Исправить опечатку в README" получает 15 баллов → выполняется сразу, без ревью.

### 2. 🧠 Структурированный мозговой штурм

**Для кого:** Сложные архитектурные решения, требующие обсуждения.

**Как работает:**
1. **Раунд 1 — Предложения:** Каждый участник предлагает свой вариант решения
2. **Раунд 2 — Критика:** Участники анализируют предложения друг друга
3. **Раунд 3 — Синтез:** Объединение лучших идей, голосование
4. **Раунд 4 — Решение:** Финальное решение + автоматическое создание ADR (документа архитектурного решения)

### 3. 📋 AI-First шаблоны

**Для кого:** Стандартизация документации проекта.

**Что включает:**
- Шаблон требований (PRD)
- Шаблон архитектурных решений (ADR)
- Шаблон эпиков и историй
- Шаблон планирования спринта

**Преимущество:** Структурированный формат, который легко обрабатывается как людьми, так и AI-ассистентами.

### 4. 📊 Менеджер задач (Task Master)

**Для кого:** Отслеживание всех задач в проекте.

**Возможности:**
- Создание задач с приоритетами
- Назначение исполнителей
- Отслеживание статуса (создана → назначена → в работе → завершена)
- История выполнения
- Комментарии и обсуждения

### 5. 📦 NPM пакет

**Для кого:** Использование фреймворка в любом проекте.

**Установка:**
```bash
npm install asmo-framework
```

**Команды:**
```bash
npx asmo analyze "Описание задачи"    # Анализ сложности
npx asmo task create "Название"        # Создать задачу
npx asmo task list                      # Список задач
```

## Уровни сложности задач

| Уровень | Баллы | Примеры | Процесс |
|---------|-------|---------|---------|
| Тривиальный | 0-20 | Опечатки, мелкие правки | YOLO Mode |
| Простой | 21-40 | Баг-фиксы, небольшие фичи | Быстрый workflow |
| Средний | 41-60 | Новые компоненты, рефакторинг | Стандартный процесс |
| Сложный | 61-80 | Интеграции, безопасность | Полное планирование |
| Корпоративный | 81-100 | Архитектура, миграции | Мозговой штурм + ревью |

## Преимущества для команды

### Для менеджеров
- Прозрачность: видно все задачи и их статусы
- Предсказуемость: система рекомендует подходящий процесс
- Документирование: все решения сохраняются в ADR

### Для разработчиков
- Скорость: простые задачи не требуют согласований
- Фокус: система подсказывает, что важно учесть
- Поддержка: готовые шаблоны для документации

### Для архитекторов
- Структура: мозговой штурм с чёткими раундами
- Консенсус: голосование и синтез идей
- Артефакты: автоматическая генерация ADR
