# ASMO -- Макеты CLI-вывода

Этот документ содержит репрезентативные макеты вывода для каждого основного сценария взаимодействия с CLI. Макеты определяют целевой формат вывода и служат справочником для реализации.

---

## Макет 7: BMAD-персонажи -- блокировка quality gate

Когда BMAD-агенты применяют принципы качества, вывод отражает их персональность:

**Amelia (QA) блокирует из-за падающих тестов:**

```
  Фаза 3/3: Тестирование (tester / Amelia)

  ЗАБЛОКИРОВАНО Amelia (QA):
   "2 из 15 тестов падают. Я не подпишу это изменение,
    пока все тесты не пройдут. Падающие тесты:
    - test/auth.test.ts: 'should reject expired tokens'
    - test/auth.test.ts: 'should handle missing claims'
    Исправьте эти тесты и перезапустите workflow."

   Статус: ЗАБЛОКИРОВАНО
   Требуется: исправить падающие тесты
```

**Bob (Product Owner) блокирует из-за двусмысленности:**

```
  Фаза 1/4: Ревью требований (product-owner / Bob)

  ЗАБЛОКИРОВАНО Bob (Product Owner):
   "Описание задачи слишком размыто. 'Улучшить страницу логина' может
    означать что угодно -- от визуальной доработки до полного переписывания. Уточните:
    1. Какую конкретную проблему испытывают пользователи?
    2. Как выглядит 'улучшенное'? (метрики, поведение, внешний вид)
    3. Есть ли существующие дизайны или спецификации?"

   Статус: ЗАБЛОКИРОВАНО
   Требуется: уточнить требования
```

**Winston (Architect) запрашивает ревью:**

```
  Фаза 1/3: Архитектурное ревью (architect / Winston)

  ПРЕДУПРЕЖДЕНИЕ от Winston (Architect):
   "Это изменение вводит новую зависимость от БД (Redis) без
    архитектурного решения (ADR). Перед продолжением:
    1. Задокументируйте обоснование Redis перед альтернативами
    2. Определите стратегию инвалидации кэша
    3. Рассмотрите режимы отказа при недоступности Redis
    Генерирую шаблон ADR для вашего ревью."

   Артефакт: adr-redis-caching.md
   Статус: ТРЕБУЕТ РЕВЬЮ
```
