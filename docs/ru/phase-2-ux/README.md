# ASMO -- UX-концепция

## Обзор

ASMO (AI System for Multiagent Orchestration) -- **CLI-first инструмент** без графического интерфейса. Все взаимодействие происходит через команду `asmo` и её подкоманды. Это осознанное проектное решение: ASMO нацелен на разработчиков, которые живут в терминале, и CLI естественно интегрируется в существующие shell-процессы, скрипты и CI/CD-пайплайны.

## Основной интерфейс

CLI предоставляет шесть подкоманд верхнего уровня:

| Команда | Назначение |
|---|---|
| `asmo run <task>` | Анализ задачи и выполнение выбранного workflow |
| `asmo suggest <task>` | Быстрая JSON-рекомендация (без выполнения) |
| `asmo analyze <task>` | Глубокий анализ сложности |
| `asmo workflow [name]` | Список всех workflow или выполнение конкретного |
| `asmo task <command>` | Управление задачами (create, list, start, complete, fail) |
| `asmo stats` | Статистика использования агентов, воркфлоу и навыков |

Каждая подкоманда поддерживает флаги для прогрессивного раскрытия -- простые вызовы дают чистый вывод, а `--verbose`, `--dry-run` и `--json` открывают детальную информацию для продвинутых пользователей.

## Двуязычная поддержка (EN + RU)

ASMO автоматически определяет язык пользователя, сканируя строку задачи на кириллические символы. Если кириллица обнаружена, вывод переключается на русский; иначе используется английский. Флаги конфигурации не нужны. Такой подход выбран, потому что основная команда двуязычна, а детекция кириллицы -- тривиальная проверка без зависимостей.

## Прогрессивное раскрытие

UX следует многоуровневой модели:

1. **Уровень 0 -- Минимальный.** `asmo run "исправь баг"` -- краткое резюме: выбранный workflow, пройденные фазы, результат.
2. **Уровень 1 -- Детальный.** `asmo run "исправь баг" --verbose` -- каждый шаг агента, LLM-вызовы, тайминги.
3. **Уровень 2 -- Только анализ.** `asmo run "исправь баг" --dry-run` -- полный план выполнения без работы.
4. **Уровень 3 -- Машиночитаемый.** `asmo suggest "исправь баг"` -- структурированный JSON для программного потребления.

## Персонажи агентов (BMAD)

Агенты ASMO -- не безликие исполнители. В рамках методологии BMAD ключевые агенты обладают выраженными личностями, которые влияют на их поведение и вывод:

| Агент | Характер | Роль |
|---|---|---|
| **Amelia** | Требовательная, блокирует при падающих тестах | Контроль качества |
| **Winston** | Методичный, настаивает на архитектурном ревью | Архитектор |
| **Bob** | Прямолинейный, блокирует при неясности | Продукт-владелец |
| **John** | Прагматичный, предпочитает работающий код | Разработчик |

Эти персонажи делают взаимодействие запоминающимся и, что важнее, естественно обеспечивают принципы качества -- Amelia не подпишет непротестированный код, а Bob отклонит размытые требования.

## Дизайн вывода

CLI-вывод использует эмодзи-индикаторы для быстрого визуального восприятия:

- `...` -- работа в процессе (спиннер/индикатор фазы)
- Галочки и кресты -- успех/неудача по фазам
- Секции анализа -- структурированные пары ключ-значение
- Прогресс -- фазовый (`Фаза 1/3: Исследование`) вместо потока логов

Результаты структурированы в секции: анализ, прогресс выполнения и финальное резюме с метриками (длительность, изменённые файлы, результаты тестов).

## Философия Zero-Config

ASMO работает из коробки. Пакет `@asmo/core` включает все 27 определений workflow, 25 файлов ролей и 55 шаблонов навыков. Никакие файлы конфигурации не нужны перед первым использованием. Для кастомизации ASMO поддерживает трёхуровневую иерархию конфигурации (проект, домашняя директория, встроенные шаблоны), где переопределения каскадируются естественным образом.

## Масштаб системы

| Ресурс | Количество |
|---|---|
| Агенты | 25 |
| Воркфлоу | 27 |
| Навыки | 55 |
| Роли | 25 (6 базовых + 15 специализированных + 4 валидационных) |
| Пакеты | `@asmo/core` + `@asmo/cli` |
