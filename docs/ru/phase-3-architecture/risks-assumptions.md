# Риски и допущения

## Реестр рисков

| # | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| R1 | Недоступность LLM API | Средняя | Высокое | Двойная LLM-стратегия с трёхрежимным fallback: Session -> API -> Heuristics. Система остаётся работоспособной (со сниженной точностью) даже при недоступности всех LLM-провайдеров. |
| R2 | Депрекация Session CLI | Низкая | Среднее | `AnthropicProvider` (API-режим) полностью реализован и протестирован как самостоятельный fallback. Миграция требует только изменения конфигурации, без изменения кода. |
| R3 | Отказы MCP-серверов | Средняя | Среднее | Приоритетная активация (P0/P1/P2) с graceful degradation. Некритичные сбои MCP не прерывают workflow. Сбои P0 изолированы по агентам. |
| R4 | Взрыв сложности workflow | Высокая | Высокое | `LearningLoop` и `MetricsOptimizer` непрерывно настраивают выбор workflow. Метрико-ориентированная обратная связь предотвращает выбор постоянно неэффективных workflow. |
| R5 | Деградация покрытия тестами | Средняя | Высокое | CI/CD-пайплайн обеспечивает минимум 80% покрытия кода. Jest запускается на каждом коммите. Отчёты покрытия отслеживаются по пакетам. |
| R6 | Дрифт конфигурации между уровнями | Средняя | Низкое | 3-уровневый каскад с явным приоритетом (проект > пользователь > встроенные). `ConfigLoader` валидирует объединённую конфигурацию по Zod-схемам перед использованием. |
| R7 | Бесконечные циклы агентов | Низкая | Высокое | `CircuitBreaker` срабатывает после 5 последовательных сбоев. `IterationManager` обеспечивает максимальное количество повторов с экспоненциальным backoff. Пофазные таймауты -- жёсткий ограничитель. |
| R8 | Исчерпание бюджета токенов | Средняя | Среднее | Маршрутизация моделей по сложности (Haiku для тривиальных, Sonnet для средних, Opus для сложных). Session-режим ($0) по умолчанию. Настраиваемые лимиты токенов на запрос и на workflow. |
| R9 | Обратно несовместимые изменения протокола MCP | Низкая | Среднее | Доступ к MCP абстрагирован через интерфейс `requestMCP()` в `BaseAgent`. Изменения протокола требуют обновления только в слое MCP bridge, а не в отдельных агентах. |
| R10 | Утечки состояния singleton в тестах | Средняя | Низкое | Каждая singleton-фабрика предоставляет функцию `resetX()`. `tests/setup.ts` сбрасывает все singleton перед каждым тестовым набором. Jest запускается с `--detectOpenHandles`. |

## Допущения

| # | Допущение | Обоснование | Fallback при невалидности |
|---|-----------|-------------|---------------------------|
| A1 | У пользователей установлен Node.js >= 18 | Необходим для ESM и современных API, используемых tsup и turbo. | Документировано в prerequisites; CLI завершается с понятной ошибкой на старых версиях. |
| A2 | Claude CLI доступен для Session-режима | Основной LLM-провайдер с $0 стоимостью. Большинство пользователей Claude Code уже его имеют. | Автоматический fallback на API-режим (требует `ANTHROPIC_API_KEY`). |
| A3 | Экосистема TypeScript остаётся стабильной | Инструменты сборки (tsup, turbo) и среда выполнения зависят от стабильности TypeScript. | Версии пакетов зафиксированы в `pnpm-lock.yaml`. |
| A4 | Протокол MCP поддерживает обратную совместимость | 8 интеграций с MCP-серверами зависят от стабильности протокола. | Слой MCP bridge абстрагирует детали протокола; обновление требуется только для bridge. |
| A5 | Anthropic API сохраняет текущую модель ценообразования | Стратегия маршрутизации моделей предполагает уровни цен Haiku < Sonnet < Opus. | Пороги маршрутизации настраиваемы и могут быть скорректированы. |
| A6 | Пользователи комфортно работают с CLI | ASMO спроектирован как CLI-first. | `asmo suggest` предоставляет JSON-вывод для потенциальной интеграции с GUI. |
| A7 | Английский и русский покрывают целевую аудиторию | Документация предоставляется на обоих языках. | Структура интернационализации поддерживает добавление новых языковых директорий. |
| A8 | SQLite достаточен для локального хранения метрик | Объём метрик ограничен частотой выполнения workflow. | Интерфейс `MetricsPersister` может быть реализован для других бэкендов. |
| A9 | Выполнение на одной машине -- основной сценарий | Нет распределённой оркестрации или координации между машинами. | Архитектура поддерживает будущее извлечение движка workflow как сервиса. |

## Внешние зависимости

### Runtime-зависимости

| Пакет | Назначение | Уровень риска |
|-------|------------|---------------|
| `anthropic` | Anthropic SDK для API-режима | Низкий -- поддерживается Anthropic |
| `zod` | Валидация схем для всех вводов и конфигураций | Низкий -- широко принят, стабильный API |
| `commander` | CLI-фреймворк для разбора аргументов | Низкий -- зрелый, минимальная поверхность API |
| `lru-cache` | In-memory кэширование для конфигурации и LLM-ответов | Низкий -- одноцелевой, стабильный |
| `yaml` | Парсинг YAML для шаблонов workflow | Низкий -- унифицирован из прежнего дублирования `js-yaml` |

### Зависимости сборки

| Пакет | Назначение | Уровень риска |
|-------|------------|---------------|
| `pnpm` | Управление workspace и зависимостями | Низкий -- отраслевой стандарт |
| `turbo` | Параллельное выполнение задач сборки | Низкий -- поддерживается Vercel |
| `tsup` | Бандлинг TypeScript в ESM/CJS | Низкий -- основан на esbuild |
| `jest` | Тестовый фреймворк с отчётами покрытия | Низкий -- широко принят |
| `typescript` | Проверка типов и компиляция | Низкий -- поддерживается Microsoft |

## Статус мониторинга и митигации

| Риск | Текущий статус | Следующее действие |
|------|----------------|-------------------|
| R1 (Недоступность LLM) | Митигирован | Трёхрежимный fallback реализован и протестирован |
| R4 (Взрыв сложности) | Митигирован | LearningLoop подключён к WorkflowEngine.initialize() |
| R5 (Покрытие тестами) | Активный | Покрытие на целевом уровне; CI-обеспечение на месте |
| R7 (Бесконечные циклы) | Митигирован | CircuitBreaker + IterationManager операционны |
| R8 (Бюджет токенов) | Частично митигирован | Маршрутизация моделей активна; лимиты на workflow в ожидании |
