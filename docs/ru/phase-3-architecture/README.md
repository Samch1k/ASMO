# Обзор архитектуры

## ASMO -- AI System for Multiagent Orchestration

ASMO -- нативная TypeScript-система мультиагентной оркестрации, автоматически маршрутизирующая задачи к специализированным AI-агентам и воркфлоу. Система организована как монорепо с двумя пакетами и использует LLM-интеллект в каждой точке принятия решений.

## Структура монорепо

| Пакет | Путь | Назначение |
|-------|------|------------|
| `@asmo/core` | `packages/core` | Основная библиотека: агенты, воркфлоу, оркестрация, интеграция с LLM |
| `@asmo/cli` | `packages/cli` | CLI-интерфейс: `asmo run`, `suggest`, `analyze`, `workflow` |

**Инструменты сборки:** pnpm (менеджер workspace) + turbo (раннер задач) + tsup (TypeScript-бандлер)
**Тестовый фреймворк:** Jest с глобальным LLM-моком в `tests/setup.ts`

## Ресурсы системы

| Ресурс | Количество |
|--------|------------|
| Агенты | 25 |
| Воркфлоу | 27 |
| Навыки | 55 |
| Роли | 25 (6 базовых + 15 специализированных + 4 валидационных) |

## Шесть архитектурных блоков

### A. Анализ задач и маршрутизация

Принимает задачу на естественном языке, оценивает сложность (0--100), выбирает оптимальный workflow, подбирает навыки и назначает агентов.

**Ключевые компоненты:** ComplexityAnalyzer, WorkflowSelector, TaskRouter, SkillMatcher, AgentRegistry

### B. Движок выполнения Workflow

Выполняет многофазные воркфлоу с логикой повторов, гейтами одобрения, автоматическими выключателями и межфазным потоком данных.

**Ключевые компоненты:** WorkflowEngine, DynamicOrchestrator, PhaseManager (6 фаз по умолчанию, настраиваемо), IterationManager, ApprovalCheckpoint (YOLO-режим), AgentExecutor, CircuitBreaker, ContextCascade, PrincipleValidators

### C. Мультиагентное взаимодействие

Четыре паттерна коллаборации для взаимодействия агентов внутри фазы workflow.

**Паттерны:** Sequential, PartySession, Brainstorm, AdversarialReview

### D. Адаптивная детекция фаз

Позволяет входить в workflow с любой фазы, анализируя существующий контекст и артефакты. Использует LLM-based ContextAnalyzer и PhaseDetector для определения оптимальной точки входа.

### E. Метрики и цикл обучения

Собирает метрики выполнения, сохраняет в SQLite и передаёт в оптимизатор для улучшения будущего выбора workflow и назначения агентов.

**Ключевые компоненты:** MetricsCollector, MetricsPersister, MetricsOptimizer, LearningLoop

### F. Конфигурация и шаблоны

Трёхуровневый каскад конфигурации с определениями ролей (JSON), навыков (SKILL.md) и шаблонами workflow (JSON).

**Ключевые компоненты:** ConfigManager, ConfigLoader, RoleManager, SkillMDLoader, PromptLoader

## Основные принципы проектирования

### Singleton-паттерн фабрик

ASMO использует 17+ singleton-фабрик по единому паттерну:

```typescript
let instance = null;
export function getX() { if (!instance) instance = new X(); return instance; }
export function resetX() { instance = null; }
```

Это гарантирует единственные экземпляры для stateful-сервисов (LLM-провайдер, метрики, конфигурация) при сохранении возможности изоляции тестов через `resetX()`.

### LLM-First интеллект

Анализ сложности, детекция фаз, выбор workflow и подбор навыков -- все используют LLM-вывод как основной механизм принятия решений, с keyword-based эвристиками в качестве оффлайн-резерва.

### Нативная TypeScript-оркестрация

`DynamicOrchestrator` заменяет прежнюю зависимость от LangGraph собственной TypeScript-реализацией, устраняя внешние зависимости от фреймворков оркестрации и обеспечивая полный контроль над потоком выполнения.

## Ключевые архитектурные решения (ADR)

| # | Решение | Обоснование |
|---|---------|-------------|
| 1 | Двойная LLM-стратегия (Session + API) | $0 по умолчанию с надёжным fallback |
| 2 | Нативная TypeScript-оркестрация | Замена LangGraph для полного контроля |
| 3 | LLM-First интеллект | Более высокая точность (~85%) против эвристик (~65%) |
| 4 | 3-уровневый каскад конфигурации | Проект, пользователь и встроенные шаблоны |
| 5 | Singleton-паттерн (17+ экземпляров) | Единообразное управление состоянием с изоляцией тестов |
| 6 | YOLO-режим (score < 30 автообход) | Быстрый путь для тривиальных задач |
| 7 | Circuit Breaker (порог 5 сбоев) | Предотвращение неконтролируемых LLM-вызовов |
| 8 | Context Cascade (межфазный) | Структурированный поток данных между фазами workflow |
| 9 | Интеграция MCP (8 серверов, P0/P1/P2) | Расширяемый доступ к инструментам для агентов |
| 10 | Principle Validators (Bob/Winston/John) | Quality gates, обеспечивающие стандарты кодирования |
