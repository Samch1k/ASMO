# ASMO v1.1.0 -- Обзор релиза

## Текущая версия

**Версия:** 1.1.0
**Статус:** Активная разработка
**Пакеты:** `@asmo/core` + `@asmo/cli`

## Сводка

ASMO (AI System for Multiagent Orchestration) -- production-ready система мультиагентной оркестрации, автоматически маршрутизирующая задачи к специализированным AI-агентам и воркфлоу. Все основные фичи реализованы и работоспособны.

## Основные возможности

| Возможность | Описание |
|-------------|----------|
| Анализ задач | LLM-powered оценка сложности и классификация по 14 категориям |
| Выполнение Workflow | 27 определённых workflow для разработки, тестирования, планирования и релиза |
| Мультиагентная коллаборация | 25 специализированных агентов с ролевой координацией |
| Адаптивная детекция фаз | LLM-powered вход в фазы -- workflow можно начать с любой фазы |
| Двойной LLM-провайдер | Session ($0 через Claude CLI) / API (плата за использование) / Heuristics (оффлайн) |
| Интеграция MCP | 8 MCP-серверов для расширенного доступа к инструментам |
| YOLO-режим | Автоодобрение задач низкой сложности (score < 30) через ApprovalCheckpoint |

## Инвентарь системы

| Ресурс | Количество |
|--------|------------|
| Агенты | 25 (6 базовых + 15 специализированных + 4 валидационных) |
| Воркфлоу | 27 |
| Навыки | 55 (по 12 категориям) |
| Роли | 25 |

## Ключевые изменения в v1.1.0

- Метаданные фазовой детекции теперь записываются в состояние для наблюдаемости
- Устранены 4 аудит-гэпа PRD: фазовая детекция, флаг `--workflow`, race condition, YOLO bypass
- IterationManager, CircuitBreaker, ContextCascade и YOLO score подключены к WorkflowEngine
- Очистка артефактов проекта, унификация YAML-зависимости (только `yaml`, удалён `js-yaml`)
- Консолидация системы навыков с 85 до 55 (-35%)
- Консолидация TEA-воркфлоу с 8 до 3 (-62%)
- Цикл обучения MetricsOptimizer подключён к `WorkflowEngine.initialize()`
