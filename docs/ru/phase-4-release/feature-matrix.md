## Ключевые платформенные фичи

| Фича | Статус | Примечания |
|------|--------|------------|
| Dual LLM Provider (Session/API/Heuristics) | Ready | Session-режим ($0) по умолчанию, API-fallback, эвристики оффлайн |
| YOLO-режим | Ready | Автоодобрение задач с оценкой сложности < 30 |
| Party Mode | Ready | Мультиагентная коллаборация на одной задаче |
| Adversarial Review | Ready | Режим ревью "найди проблему" для критического кода |
| Адаптивная детекция фаз | Ready | LLM-powered пропуск фаз, вход в workflow в середине |
| Context Cascade | Ready | Межфазный поток данных, без потери контекста между агентами |
| Circuit Breaker | Ready | Порог 5 сбоев, предотвращение каскадных отказов агентов |
| Iteration Manager | Ready | Управление циклами итеративного уточнения внутри workflow |
| BMAD-персонажи | Ready | 4 режима персональности для настройки поведения агентов |
| Двуязычная поддержка (EN/RU) | Ready | Автодетекция кириллицы, полная поддержка русского языка |
| Метрики и обучение | Ready | SQLite-персистенция, цикл обучения MetricsOptimizer |
| Интеграция MCP | Ready | 8 MCP-серверов для расширенного доступа к инструментам |
| Zod-валидация | Ready | Все вводы валидируются по Zod-схемам |
| Санитизация ввода | Ready | 10 проверок: UTF-8, контрольные символы, null-байты и др. |
| Keyword-анализ | Ready | 14 категорий, централизованный движок классификации |
| Режим `--dry-run` | Ready | Превью плана выполнения без запуска |
| Режим `--verbose` | Ready | Детальное логирование активности агентов |
| Переопределение `--workflow` | Ready | Принудительный выбор конкретного workflow |
| Режим `--no-llm` | Ready | Только эвристики, оффлайн, ~65% точности |
| Режим `--use-api` | Ready | Принудительный API-режим (плата за токен) |
