## v1.1.0 (Текущая) -- Февраль 2026

### Основные моменты

Этот релиз фокусируется на наблюдаемости, надёжности и консолидации системы. Ключевые улучшения включают метаданные фазовой детекции для отладки поведения workflow, устранение аудит-гэпов PRD и значительное снижение сложности системы через консолидацию навыков и workflow.

### Изменения

**Наблюдаемость и фазовая детекция**
- Метаданные фазовой детекции теперь записываются в состояние workflow, обеспечивая полную наблюдаемость того, как ASMO принимает решение о входе в фазу
- Метаданные `phaseDetection` захватываются в состоянии для каждого выполнения workflow

**Устранение аудит-гэпов PRD (4 исправления)**
- Фазовая детекция: адаптивное присоединение к фазе теперь корректно обрабатывает все edge-кейсы
- Флаг `--workflow`: ручное переопределение workflow работает сквозно от CLI до WorkflowEngine
- Race condition: конкурентная инициализация workflow больше не порождает undefined-состояние
- YOLO bypass: ApprovalCheckpoint корректно пропускает одобрение для задач низкой сложности (score < 30)

**Интеграция WorkflowEngine**
- IterationManager подключён к WorkflowEngine для циклов итеративного уточнения
- CircuitBreaker интегрирован с порогом 5 сбоев для предотвращения каскадных отказов агентов
- ContextCascade обеспечивает межфазный поток данных без потери контекста
- YOLO score передаётся в ApprovalCheckpoint для автоматических решений обхода
- Цикл обучения MetricsOptimizer подключён к `WorkflowEngine.initialize()`

**Консолидация системы**
- Система навыков консолидирована с 85 до 55 навыков (-35%), удалено дублирование и объединены перекрывающиеся возможности
- TEA-воркфлоу консолидированы с 8 до 3 (-62%), без потери покрытия
- Очистка артефактов проекта и файлов бэкапов
- YAML-зависимость унифицирована до пакета `yaml` (удалён дубликат `js-yaml`)

## v1.0.0 -- Январь 2026

### Основные моменты

Начальный релиз ASMO с основными возможностями оркестрации.

**Основные фичи**
- Мультиагентная оркестрация задач с автоматическим выбором workflow
- 14-категорийный keyword-анализ для классификации задач
- Двойной LLM-провайдер: Session ($0 через Claude CLI) и API (плата за токен)
- Heuristics-fallback для оффлайн-работы
- YOLO-режим для автоматического одобрения тривиальных задач
- Система персонажей BMAD с 4 режимами
- Двуязычная поддержка (английский и русский) с автодетекцией кириллицы
- Интеграция MCP с 8 серверами
- Zod-based валидация ввода с 10 проверками безопасности
- Персистенция метрик в SQLite

**Архитектура**
- Структура монорепо с `@asmo/core` и `@asmo/cli`
- Класс BaseAgent с `callLLM()`, `callLLMForJSON()`, `createResult()`, `createArtifact()`, `requestMCP()`
- Singleton-паттерн фабрик для экземпляров сервисов
- ConfigLoader для ролей (JSON), навыков (SKILL.md), workflow (JSON-шаблоны)
- Фабричный метод `WorkflowEngine.create()`
