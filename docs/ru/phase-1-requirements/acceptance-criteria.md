# Критерии приемки

Этот документ определяет тестируемые критерии приемки для каждого функционального
требования (FR). Каждый критерий имеет уникальный идентификатор, описание и
приоритет.

---

## AC-1.x: Анализ задач и маршрутизация (FR-1)

| ID | Критерий | Приоритет |
|----|---------|-----------|
| AC-1.1 | `ComplexityAnalyzer` принимает строку задачи и возвращает оценку 0--100 и уровень (trivial/simple/medium/complex/enterprise) | P0 |
| AC-1.2 | LLM-классификация обеспечивает точность >= 77% на тестовом наборе из 100 задач | P0 |
| AC-1.3 | Эвристический анализ по ключевым словам обеспечивает точность >= 65% при отключенном LLM | P0 |
| AC-1.4 | `WorkflowSelector` выбирает корректный воркфлоу для каждой из 14 категорий ключевых слов | P0 |
| AC-1.5 | `TaskRouter` направляет задачу в выбранный воркфлоу менее чем за 2 секунды | P0 |
| AC-1.6 | `SkillMatcher` разрешает зависимости навыков из `skill-dependencies.json` | P1 |
| AC-1.7 | Модификаторы влияния (high_risk, data_impact, security_impact, performance_impact) корректно изменяют оценку | P1 |
| AC-1.8 | Ввод валидируется по 10 проверкам (пустота, длина, UTF-8, управляющие символы, нормализация) | P0 |
