## Философия ASMO

### От ассистента к команде

Традиционные AI-ассистенты отвечают на отдельные запросы. ASMO превращает AI в автономную команду разработки:

- **Несколько специализированных агентов** работают вместе
- **Workflows оркестрируют** сложные многоступенчатые задачи
- **Навыки подбираются** автоматически под требования
- **Quality gates** обеспечивают качество результата

### Адаптивный интеллект

ASMO не использует универсальные подходы:

1. **Анализ** — оценка сложности задачи (0-100)
2. **Выбор** — подбор подходящего workflow
3. **Выполнение** — агенты работают последовательно или параллельно
4. **Валидация** — проверка качества на каждом этапе

## Как это работает

1. **Задача поступает** — описание на естественном языке
2. **ComplexityAnalyzer** оценивает сложность
3. **WorkflowEngine** выбирает подходящий workflow
4. **Агенты** выполняют шаги workflow
5. **TaskManager** отслеживает прогресс
6. **Quality gates** валидируют результат

## Пример потока

```typescript
// 1. Описание задачи
const task = "Добавить OAuth2 аутентификацию в API"

// 2. Анализ сложности (автоматически)
// Оценка: 65, Уровень: Complex, Агенты: architect, security, developer

// 3. Выбор workflow (автоматически)
// Выбран: security_audit + feature_implementation

// 4. Выполнение
const result = await engine.execute(task)

// 5. Результат со всеми артефактами
// - ADR для подхода к auth
// - Оценка безопасности
// - Код реализации
// - Тесты
```
