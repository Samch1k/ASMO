# Adversarial Review (Критический код-ревью)

Adversarial Review — это подход к критическому код-ревью, при котором ревьюер **ОБЯЗАН** найти проблемы. Эта техника обеспечивает тщательный анализ и предотвращает поверхностные одобрения.

## Обзор

Традиционный код-ревью часто страдает от:
- Формальных одобрений без анализа
- Поверхностного просмотра кода
- Предвзятости подтверждения

Adversarial Review решает эти проблемы, требуя от ревьюера активного поиска проблем с эскалацией глубины анализа.

## Как это работает

```
┌─────────────────────────────────────────────────────────┐
│                   Adversarial Review                     │
├─────────────────────────────────────────────────────────┤
│  Попытка 1: Стандартный ревью                           │
│  ├─ Найдены проблемы? → Вернуть проблемы               │
│  └─ Нет проблем? → Эскалация                           │
│                                                          │
│  Попытка 2: Глубокий анализ                             │
│  ├─ "Ищите тщательнее edge cases"                      │
│  ├─ Найдены проблемы? → Вернуть проблемы               │
│  └─ Нет проблем? → Эскалация                           │
│                                                          │
│  Попытка 3: Предполагаем, что проблемы есть            │
│  ├─ "Проблемы ДОЛЖНЫ быть - найдите их"                │
│  ├─ Найдены проблемы? → Вернуть проблемы               │
│  └─ Нет проблем? → Одобрить с предупреждением          │
└─────────────────────────────────────────────────────────┘
```

## Использование

```typescript
import { AdversarialReviewSession, AdversarialReviewerAgent } from '@asmo/core'

// Создаём сессию и ревьюера
const session = new AdversarialReviewSession({
  minIssuesRequired: 1,
  maxRetries: 3
})
const reviewer = new AdversarialReviewerAgent()

// Выполняем ревью
const artifact = {
  id: 'auth-module',
  type: 'code',
  content: `
    function authenticate(user, password) {
      return db.query('SELECT * FROM users WHERE name=' + user)
    }
  `,
  metadata: { language: 'javascript' }
}

const result = await session.executeReview(artifact, reviewer)

console.log('Найдено проблем:', result.issues.length)
console.log('Одобрено:', result.approved)
result.issues.forEach(issue => {
  console.log(`- [${issue.severity}] ${issue.title}`)
})
```

## Уровни эскалации

Система использует всё более строгие промпты:

### Уровень 1: Стандартный ревью
> "Проведите критический анализ артефакта. Найдите проблемы с качеством кода, безопасностью, производительностью или поддерживаемостью."

### Уровень 2: Глубокий анализ
> "Предыдущий ревью не нашёл проблем. Ищите тщательнее. Рассмотрите edge cases, обработку ошибок, уязвимости безопасности и потенциальные race conditions."

### Уровень 3: Предполагаем наличие проблем
> "Идеального кода не бывает. Проблемы ДОЛЖНЫ быть. Найдите их: уязвимости безопасности, узкие места производительности, проблемы поддерживаемости, отсутствие обработки ошибок, edge cases."

## Конфигурация

```typescript
const config = {
  adversarialReview: {
    enabled: true,
    minIssuesRequired: 1,    // Минимум проблем для поиска
    maxRetries: 3            // Количество попыток эскалации
  }
}
```

## Категории проблем

| Категория | Описание |
|-----------|----------|
| `security` | Уязвимости безопасности (XSS, SQL injection и т.д.) |
| `performance` | Узкие места производительности |
| `maintainability` | Проблемы качества и поддерживаемости кода |
| `logic` | Логические ошибки или баги |
| `edge-case` | Отсутствие обработки граничных случаев |
| `error-handling` | Недостаточная обработка ошибок |

## Уровни критичности

| Уровень | Описание |
|---------|----------|
| `critical` | Требует немедленного исправления |
| `major` | Должно быть исправлено до мерджа |
| `minor` | Желательно исправить, не блокирует |
| `suggestion` | Опциональное улучшение |
