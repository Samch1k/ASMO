# Навыки

Навыки — это конкретные способности, которые агенты используют для выполнения задач.

## Что такое навык?

Навык определяет:
- **Способность** — что навык делает
- **Сложность** — уровень трудности (базовый — экспертный)
- **Требования** — другие навыки или инструменты
- **Оценка времени** — ожидаемое время выполнения

## Категории навыков

ASMO включает **85 навыков** в **12 категориях**:

| Категория | Кол-во | Примеры |
|-----------|--------|---------|
| 🏗️ Архитектура | 7 | System design, ADR creation |
| 👨‍💻 Разработка | 8 | Code writing, feature implementation |
| 🐛 Отладка | 7 | Bug diagnosis, root cause analysis |
| 🧪 Тестирование | 8 | Unit testing, E2E testing |
| ⚡ Производительность | 8 | Profiling, optimization |
| 🎨 UI/UX Дизайн | 7 | Styling, accessibility |
| 🚀 DevOps | 6 | Deployment, CI/CD |
| 💼 Бизнес | 5 | Requirements, KPIs |
| 🎯 Продукт | 4 | Strategy, prioritization |
| 📋 Управление проектом | 4 | Coordination, tracking |
| 📊 Анализ | 4 | BMad analysis |
| ⭐ Суперсилы | 5 | Advanced workflows |

## Уровни сложности

| Уровень | Трудность | Примеры навыков |
|---------|-----------|-----------------|
| 🟢 Базовый | 1-3 | Простые исправления, документация |
| 🟡 Средний | 4-6 | Code writing, тестирование |
| 🟠 Продвинутый | 7-8 | Оптимизация производительности |
| 🔴 Экспертный | 9-10 | Архитектурные решения |

## Подбор навыков

ASMO автоматически подбирает навыки:

### По ключевым словам

```typescript
// Задача: "Исправить баг аутентификации"
// Подобранные навыки: bug_diagnosis, root_cause_analysis, hotfix_generation
```

### По алиасам

```typescript
// Задача: "Написать unit тесты"
// Подбирает "unit_testing" через алиас "unit tests"
```

### По композиции

Навыки комбинируются для сложных задач:

```typescript
// Задача: "Добавить новый API endpoint"
// Скомпонованные навыки:
// - architecture_decisions (требуется для api_design)
// - api_design
// - code_writing
// - unit_testing
```

## Зависимости навыков

Некоторые навыки требуют других:

```
architecture_decisions
         ↓
    adr_creation
         ↓
  technology_evaluation
```
