# Границы проекта

Этот документ определяет границы того, что ASMO обрабатывает и что следует
делать напрямую без оркестрации.

## В скоупе -- Использовать ASMO, когда

### Индикаторы сложности

Следующие характеристики сигнализируют о том, что задача выигрывает от
мультиагентной оркестрации:

- **Множество файлов** -- необходимо изменить файлы в разных частях кодовой базы.
- **Множество аспектов** -- затронуты код + тесты + документация одновременно.
- **Архитектурные решения** -- требуются до начала реализации.
- **Сквозные аспекты** -- затронуты безопасность, производительность или обсервабилити.

### Специализированная работа

ASMO предоставляет выделенных агентов и воркфлоу для доменных задач:

| Домен | Примеры | Соответствующие воркфлоу |
|-------|---------|------------------------|
| **Безопасность** | Аутентификация, авторизация, исправление уязвимостей | `security_audit` |
| **Производительность** | Оптимизация, профилирование, стратегии кэширования | `performance_optimization`, `performance_investigation` |
| **Архитектура** | Системный дизайн, рефакторинг, проектирование API | `architecture_design`, `api_design`, `code_refactoring` |
| **Базы данных** | Миграции, изменения схемы, оптимизация запросов | `database_migration` |
| **Тестирование** | Тестовые наборы, E2E-тесты, улучшение покрытия | `comprehensive_testing`, `tea_*` воркфлоу |
| **DevOps** | CI/CD, деплой, инфраструктура | Воркфлоу, связанные с деплоем |

### Мультиагентные задачи

Задачи, требующие координации между несколькими специализированными ролями:

| Паттерн задачи | Вовлеченные агенты | Воркфлоу |
|----------------|-------------------|----------|
| Реализация фичи | Architect + Developer + Tester | `feature_implementation_full` |
| Расследование и исправление бага | Debugger + Developer + Tester | `bug_fix_workflow` |
| Аудит безопасности | Security Specialist + Tester | `security_audit` |
| Оптимизация производительности | Optimizer + Developer | `performance_optimization` |
| Adversarial код-ревью | Code Reviewer + Developer | `adversarial_review_workflow` |

### Быстрая проверка

Выполните следующую команду для автоматической рекомендации:

```bash
asmo suggest "<ваша задача>"
```

Возвращает JSON с `useAsmo: true/false` и обоснованием на основе анализа
ключевых слов и сложности.

## Вне скоупа -- Работать напрямую

Следующие задачи следует выполнять без оркестрации ASMO. Использование ASMO
для них создает ненужные накладные расходы:

| Тип задачи | Примеры | Почему напрямую лучше |
|-----------|---------|---------------------|
| **Простые опечатки** | Исправить неправильно написанное имя переменной | Одно изменение, координация не нужна |
| **Однострочные изменения** | Обновить номер версии, переключить флаг | Накладные расходы превышают пользу |
| **Правки документации** | Исправить битую ссылку, обновить абзац | Если не требуется полная переработка |
| **Быстрые вопросы** | «Что делает эта функция?» | Воркфлоу не нужен, просто прочитать код |
| **Исследование файлов** | «Покажи структуру проекта» | Стандартная задача CLI / редактора |

### Правило большого пальца

Если задачу можно выполнить одним промптом с изменением одного файла --
делайте напрямую. Если требуется планирование, несколько файлов или доменная
экспертиза -- используйте ASMO.

## Эволюция скоупа

По мере развития ASMO скоуп может расшириться:

- **Интеграция с IDE** -- Расширение для VS Code для оркестрации прямо
  в редакторе.
- **Интеграция с CI/CD** -- Автоматические контрольные точки качества
  в воркфлоу pull request-ов.
- **Командная оркестрация** -- Координация между несколькими экземплярами
  ASMO разных разработчиков.

Это не обязательства по доставке. Они представляют потенциальные будущие
направления, отслеживаемые в дорожной карте проекта.
