# ASMO -- Видение / Одностраничник

## Что такое ASMO?

**ASMO (AI System for Multiagent Orchestration)** превращает ИИ из «помощника»
в **автономную команду**. Это многоагентная система оркестрации, построенная
на TypeScript, которая реализует методологию BMAD (Breakthrough Method of Agile AI Driven
Development).

Вместо одного ИИ, отвечающего на промпты, ASMO координирует **25
специализированных агентов** через **27 структурированных воркфлоу**, используя
**55 навыков** и **25 ролей** для доставки результатов продакшн-уровня.

## Ключевая идея

Современные задачи разработки -- реализация фич, аудит безопасности, оптимизация
производительности -- не являются одношаговыми проблемами. Они требуют
планирования, проектирования, кодирования, тестирования и ревью. ASMO
моделирует эту реальность, направляя каждую задачу через правильную комбинацию
агентов и воркфлоу -- так же, как это сделала бы хорошо организованная
инженерная команда.

## Ключевые возможности

| Возможность | Описание |
|-------------|---------|
| **Адаптивный выбор воркфлоу** | Автоматически подбирает лучший воркфлоу для любой задачи |
| **Интеллектуальная маршрутизация** | Направляет работу наиболее квалифицированным агентам на основе сложности и домена |
| **Автоматический анализ сложности** | Оценивает задачи 0--100 по пяти уровням (trivial, simple, medium, complex, enterprise) |
| **Dual LLM Strategy** | Session ($0 через Claude CLI) -> API (pay-per-token) -> Heuristics (офлайн) |
| **Мультиагентная коллаборация** | Последовательный, параллельный, брейншторминг (Party Mode) и adversarial-паттерны |

## Ключевые отличия

- **YOLO Mode** -- Задачи низкой сложности автоматически одобряются и
  выполняются без участия человека, снижая задержку до менее 60 секунд.
- **Party Mode** -- Несколько агентов проводят брейншторм одновременно,
  создавая более качественные архитектурные решения и ADR.
- **Adversarial Review** -- Выделенный агент-ревьюер проверяет код с
  нарастающей строгостью, выявляя проблемы, которые обычное ревью пропускает.
- **Adaptive Phase Detection** -- Воркфлоу могут быть присоединены на любой
  фазе на основе существующих артефактов, избегая повторной работы.
- **$0 стоимость разработки** -- Session-провайдер по умолчанию использует
  Claude CLI без дополнительных затрат на API.

## Принципы проектирования

ASMO основан на методологии BMAD:

1. **Business-driven** -- Каждый воркфлоу привязан к реальному бизнес-результату.
2. **Multi-agent** -- Специализированные агенты сотрудничают, а не один генералист делает все.
3. **Adaptive** -- Система адаптируется к сложности задачи, существующим артефактам и доступным ресурсам.
4. **Developer-first** -- Native TypeScript, минимальные зависимости, CLI-native интерфейс.

## Позиционирование

ASMO -- это **легковесная, нативная TypeScript-альтернатива** фреймворкам
LangGraph, CrewAI и AutoGen. Где эти системы требуют Python-рантайм, сложные
графы зависимостей или облачную инфраструктуру, ASMO работает локально
из единственной команды `asmo run`.
